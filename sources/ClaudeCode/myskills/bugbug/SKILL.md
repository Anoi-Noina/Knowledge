# bugbug — バグ修正 → 設計書同期

## 目的
バグ修正後、設計書が更新されずに放置されるのを防ぐ。修正完了・ユーザー確認後に、設計書との整合性を確認し必要なら修正する。

## 手順

1. **バグ修正**
   - 依頼されたバグを調査し、原因を特定して修正する。
   - 通常のバグ修正フロー（再現確認 → 原因特定 → 修正 → 動作確認）に従う。

2. **ユーザー確認を待つ**
   - 修正内容をユーザーに提示し、明示的な確認（「直った」「OK」「問題ない」等）を得るまで、次のステップに進まない。

3. **設計書の確認**
   - ユーザーの確認が取れたら、リポジトリ内で関連する設計書を探す（例: `設計書`, `design` 等を含むファイル名や、対象モジュール近辺のドキュメント）。
   - 見つからない場合は、憶測で探し続けたり勝手に新規作成したりせず、ユーザーに設計書の場所を確認する。

4. **差分の判断**
   - 見つかった設計書の内容と今回の修正内容を突き合わせ、記載内容に矛盾や古い情報がないか確認する。
   - 修正が設計書の記載範囲外の軽微なもの（誤字修正、実装内部だけの話など）であれば、設計書の更新は不要と判断してよい。

5. **必要であれば更新**
   - 設計書の記載が実態と食い違うようになった場合のみ更新する。
   - 更新内容をユーザーに提示し、確認を得る。

## 禁止事項

- ユーザーの確認前に設計書を触らない。
- 設計書が見つからない場合に、憶測で作成したり無視したりしない。必ずユーザーに確認する。
